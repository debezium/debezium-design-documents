# Introduction

An ideal GSoC proposal should include the following:

1. link to the **the Zulip message** that introduce you
2. Link to **code contributions**, such as a PRs, you've made previously
3. Full description of your proposed project, including roadmap with goals and strech goals
4. Your schedule during the GSoC period and any prior committments that might affect your availability (school exams, other work committments, travel, etc)

Example GSoC proposal template:


# Debezium: The thing I want to do this summer

Note: Make sure to include the sub-org name in the title both in Google's system and in your document.

## About me

1. Name (and nicknames like your github and irc usernames)
2. University / program / year / expected graduation date
3. Contact info (email, phone, etc.)
4. Time zone
5. Link to a resume (if you want)

## Code Contribution

- Link to pull requests goes here.
    - the PR must be code submitted in Debezium organization.
    - It must represent your own work, although you can have help from developers to improve it.
    - It must be publicly visible to your mentors and org admins.

## Project Information

### Abstract

Short description of your project, 5-10 sentences.

## Why this project?

What interests you in this GSoC project and our organization?

### Technical Description

More detailed description of your project.
**Should** include all technical details of the projects for example:
- Overview of system design with diagrams (we suggest [excalidraw](https://excalidraw.com/))
- APIs and code examples to explain the Developer Experience
- Tradeoff of your proposed solution
- references to external documentation

For this section, it is important to show if you had previous conversations with your mentors and have a sufficient understanding of the problem and a plan to address it during your project. The conversations must be done in public and in the official Debezium channel for GsoC #community-gsoc. Submit a PR for review.

Example of design document are the following:

- [Quarkus Debezium Extension](https://github.com/debezium/debezium-design-documents/blob/main/DDD-12.md)
- [Read-only incremental snapshots for other relational connectors](https://github.com/debezium/debezium-design-documents/blob/main/DDD-12.md)
